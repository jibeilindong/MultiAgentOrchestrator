# Agent 模板库与 SOUL 纯净原则

## 目标

这套模板系统把 agent 模板拆成两个层面：

- 管理层：用于模板分类、排序、筛选、导入导出、收藏和运营管理。
- SOUL 层：只表达 agent 的角色定位、使命、能力、职责、流程、协作边界与成功标准。

核心约束只有一条：`SOUL.md` 里不允许出现模板管理信息。

## 模板结构

模板在代码中由两部分组成：

- `AgentTemplateMeta`
  - 管理字段，如 `id`、分类、摘要、能力标签、模板标签、颜色、排序和推荐状态。
- `AgentTemplateSoulSpec`
  - 功能字段，如角色定位、核心使命、核心能力、输入要求、工作职责、工作流程、输出要求、协作边界、行为边界和成功标准。

最终 `SOUL.md` 只由 `AgentTemplateSoulSpec` 渲染生成。

## 内置模板体系

内置模板按两大主族组织：

- 功能型
  - 学习、训练、测试
  - 监督、考察
  - 日志分析
  - 记忆优化
  - HR、招聘与工作流
- 作业型
  - 文档类
  - 视频类
  - 代码类
  - 图片类

现有模板已经统一并入这套分类体系，后续新增模板也应先归类，再补 SOUL 内容。

## 管理系统能力

当前模板库已经支持：

- 内置模板覆盖与自定义模板扩展
- 收藏、最近使用、标签筛选与自定义排序
- 基于当前 agent 名称、身份、描述、能力和 SOUL 内容做模板推荐
- 对缺失的 SOUL 必填字段执行保守式自动补齐，不覆盖已有内容
- 在校验扫描页对当前筛选结果执行批量自动补齐
- 对 `SOUL.md` 中泄漏的模板管理词汇提供清理建议，并支持单个或批量清理
- 在执行清理前预览字段级差异，查看清理前后内容和命中的管理词
- JSON 模板导入导出
- 现有 `SOUL.md` 反向导入为模板，或载入到当前草稿
- 纯净 `SOUL.md` 单个导出与批量导出
- 模板校验扫描

持久化文件位置：

- `TemplateLibrary/agent-template-library.json`

这个文件保存的是模板管理快照，不是最终给 agent 使用的 `SOUL.md`。

## 校验规则

模板保存和扫描时会检查：

- SOUL 必填字段是否为空
- 列表字段是否过长
- `SOUL.md` 是否泄漏管理词汇

当前会拦截的典型泄漏词包括：

- `功能型`
- `作业型`
- `模板管理`
- `输入协议`
- `输出协议`
- `运行规则`

如果这些词出现在 `SOUL.md` 中，模板会被标为错误。

## 扩展原则

后续扩模板时，建议遵循以下顺序：

1. 先确定模板属于哪个主族和子类。
2. 再补管理字段，方便筛选、排序和复用。
3. 最后编写精简、准确、完整的 SOUL 内容。
4. 保存后通过校验扫描，再导出 `SOUL.md` 交付使用。
