# Multi-Agent-Flow 嵌套子流程使用指南

本文档介绍 Multi-Agent-Flow 中的嵌套子流程功能，用于创建复杂的多层级工作流。

## 概述

嵌套子流程（Nested Subflow）允许在工作流中创建多层级结构，将复杂的业务逻辑分解为可重用的子模块。

### 核心概念

| 概念 | 说明 |
|------|------|
| 子流程节点 | 工作流中的特殊节点类型，用于引用子工作流 |
| 嵌套层级 | 表示子流程的深度（0=顶层） |
| 父工作流 | 包含子流程节点的主工作流 |
| 子工作流 | 被父工作流引用的独立工作流 |

### 节点类型

工作流节点分为四种类型：

| 类型 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| Agent | 👤 人物圆形 | 蓝色 | 执行具体任务的智能体 |
| Start | ▶️ 播放圆形 | 绿色 | 工作流起始节点 |
| End | ⏹️ 停止圆形 | 红色 | 工作流结束节点 |
| Subflow | 📄 文档堆叠 | 紫色 | 嵌套的子工作流 |

## 使用指南

### 创建子流程节点

1. 在工作流编辑器的工具栏中，点击添加节点按钮
2. 在节点类型选择器中，选择 **Subflow**（紫色图标）
3. 子流程节点出现在画布上，显示为紫色方块

### 编辑子流程

1. **双击**子流程节点，或右键选择 **Edit Subflow**
2. 弹出子流程编辑器面板
3. 可以选择：
   - **创建新子流程**：输入名称后创建空白子流程
   - **选择现有子流程**：从已有列表中选择

### 查看子流程详情

子流程编辑器面板显示以下信息：

- 子流程名称
- 节点数量
- 连接数量
- 当前嵌套层级
- 节点预览列表

### 删除子流程

1. 在子流程编辑器面板中
2. 点击 **Delete Subflow** 按钮
3. 确认后删除子流程及其所有节点

## 嵌套层级

系统支持多层嵌套，层级显示在子流程编辑器中：

```
Level 0: 主工作流
  └── Level 1: 子流程1
        └── Level 2: 子流程2
              └── Level 3: 子流程3 (最大层级)
```

### 嵌套层级限制

- 最大嵌套深度：**3 层**
- 超过最大层级时，系统会阻止创建新的子流程节点
- 层级越深，渲染和执行的开销越大

### 层级可视化

子流程节点会根据嵌套层级显示不同的边框样式：
- Level 1: 单线边框
- Level 2: 双线边框
- Level 3: 粗线边框

## 数据流

### 父到子数据传递

```
父工作流 → [输入数据] → 子流程节点 → 子工作流
```

### 子到父数据返回

```
子工作流 → [输出数据] → 子流程节点 → 父工作流
```

### 执行顺序

执行主工作流时，遇到子流程节点会：
1. 进入子工作流执行
2. 等待子工作流完成
3. 收集输出结果
4. 继续执行父工作流的后续节点

## 最佳实践

### 何时使用子流程

| 场景 | 推荐 |
|------|------|
| 重复使用的流程模块 | ✅ 使用子流程实现复用 |
| 复杂业务流程分解 | ✅ 拆分为多个子流程 |
| 条件分支逻辑 | ⚠️ 考虑使用 Agent 条件判断 |
| 简单线性流程 | ❌ 不需要子流程 |

### 子流程设计原则

1. **单一职责**：每个子流程只完成一个特定任务
2. **清晰命名**：使用描述性的名称，如 "用户认证子流程"
3. **适当大小**：单个子流程建议 3-15 个节点
4. **层级控制**：避免过深的嵌套（建议最多 2 层）

### 常见模式

#### 模式一：模块化复用

```
主工作流
  ├── 节点A
  ├── [子流程: 数据处理]
  ├── 节点B
  └── [子流程: 数据处理]  ← 复用同一子流程
```

#### 模式二：渐进细化

```
主工作流
  └── [子流程: 订单处理]
        ├── [子流程: 库存检查]
        ├── [子流程: 支付处理]
        └── [子流程: 物流配送]
```

#### 模式三：错误处理

```
主工作流
  ├── 节点A
  ├── [子流程: 核心业务]  ← 主流程
  │     └── [子流程: 错误处理]  ← 异常分支
  └── 节点B
```

## 快捷操作

| 操作 | 方法 |
|------|------|
| 创建子流程节点 | 工具栏 → 选择 Subflow 类型 |
| 编辑子流程 | 双击节点 或 右键 → Edit Subflow |
| 查看嵌套层级 | 子流程编辑器面板中显示 |
| 删除子流程 | 编辑器 → Delete Subflow |

## 故障排查

### 子流程节点无法创建
- **原因**：已达到最大嵌套层级（3层）
- **解决**：重新设计工作流结构，减少嵌套深度

### 子流程不执行
- **原因**：子流程中缺少 Start 或 End 节点
- **解决**：确保子流程包含完整的起止节点

### 数据传递失败
- **原因**：父节点和子流程的输入/输出字段不匹配
- **解决**：检查两者的数据结构是否兼容

### 子流程编辑器无法打开
- **原因**：子流程节点未正确关联到子工作流
- **解决**：重新创建子流程或检查节点配置
